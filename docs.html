Documentation

Everything you need to run Vigil — from quick install to MCP server configuration.

Installation

Git Clone (recommended)

git clone https://github.com/vigil-agency/vigil.git
cd vigil
cp .env.example .env
npm install
npm start
# → http://localhost:4100 (admin / admin)

One-Liner

curl -fsSL https://raw.githubusercontent.com/vigil-agency/vigil/main/install.sh | bash

Prerequisites

  • Node.js 18+ (22 recommended)
  • Scanners (optional): nmap, nuclei, trivy, nikto, openssl, dig, whois
  • Docker CLI (optional, for container scanning)
  • PostgreSQL 17 (optional — works without it)
  • AI CLI (optional): claude or codex

Configuration

Copy .env.example to .env and customize:

VariableDefaultDescription
VIGIL_PORT4100Server port
VIGIL_USERadminDefault admin username
VIGIL_PASSadminDefault admin password
DATABASE_URL—PostgreSQL connection (optional)
AI_PROVIDER—claude-cli or codex
ENCRYPTION_KEYauto32-byte hex for credential vault
DOCKER_HOST—Docker socket for container scanning

Docker

git clone https://github.com/vigil-agency/vigil.git
cd vigil && cp .env.example .env
docker compose up -d
# Starts Vigil + PostgreSQL 17

The Docker image includes all scanners pre-installed (nmap, nuclei, trivy, nikto, openssl, dig/whois).

MCP Server

Built-in Model Context Protocol server at POST /mcp (Streamable HTTP transport).

Connect from Claude Desktop

{
  "mcpServers": {
    "vigil": {
      "url": "http://localhost:4100/mcp",
      "transport": "streamable-http"
    }
  }
}

Architecture

server.js              → Express + Socket.IO, auth, intervals
routes/  (27 modules)  → REST API endpoints
lib/     (17+ modules) → Scanner wrappers, AI, RBAC, crypto
data/                  → JSON file stores (no DB required)
public/                → Vanilla JS frontend (30 views)
  css/                 → Vigil Dark theme (glass treatment)
  js/views/            → Self-registering ViewRegistry modules

6 npm dependencies: express, socket.io, pg, node-pty, multer, @modelcontextprotocol/sdk

Auth & RBAC

  • PBKDF2 password hashing
  • Cookie sessions (vigil_session) or Bearer token
  • Optional TOTP 2FA
  • 3 roles: admin (full), analyst (scan + view), viewer (read-only)
  • AES-256-GCM credential vault

Vigil Dark Theme

Glass treatment with backdrop blur. Signal system:

  • Orange (#ff6b2b) — accent, primary actions, brand
  • Cyan (#22d3ee) — secure, healthy, passing
  • Red (#ef4444) — threats, vulnerabilities, critical
